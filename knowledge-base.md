# 工业视觉软件测试知识库

## 1. 软件架构与功能模块

### 1.1 整体架构
- **左侧面板**：工具树结构，显示物料方案、任务流程、相机配置、图像采集和图像处理工具
- **中间面板**：画布区域，显示相机拍照图片，支持来源图、预处理图、完成图切换
- **右侧面板**：参数配置面板，根据选择的工具显示对应的配置参数

### 1.2 核心功能模块
- **物料方案管理**：创建和管理不同的物料检测方案
- **任务流程配置**：配置完整的检测任务流程
- **相机配置**：管理CCD0、CCD1等相机设备
- **图像采集**：配置相机拍照参数，注册基准图像
- **图像处理工具**：各种检测和分析工具的配置和使用

## 2. 图像处理工具知识库

### 2.1 面积检测工具
**功能描述**：
- 用于检测图像中特定区域的面积大小
- 支持多种形状的ROI设置（矩形、圆形、多边形、自由曲线）
- 可设置面积阈值进行合格/不合格判断
- 支持单位换算（像素、毫米、平方厘米等）

**参数配置**：
- **工具名称**：自定义工具标识
- **图像来源**：选择图像采集或上游工具输出
- **位置校准**：选择位置校准工具输出
- **ROI类型**：矩形/圆形/多边形/自由曲线
- **ROI参数**：根据ROI类型设置相应参数
- **面积计算方法**：像素计数/轮廓分析
- **面积阈值**：最小面积、最大面积
- **单位选择**：像素、毫米、平方厘米
- **结果输出**：面积值、合格状态

**测试重点**：
- ROI设置的准确性和灵活性
- 面积计算的精度验证
- 不同形状和复杂度的区域处理
- 边界条件处理（极小面积、极大面积）
- 单位换算的准确性

**常见问题**：
- ROI设置不当导致检测区域错误
- 光照条件变化影响面积计算精度
- 复杂背景干扰导致面积误算
- 边界模糊导致轮廓提取不准确

### 2.2 瑕疵检测工具
**功能描述**：
- 用于检测产品表面的瑕疵、缺陷
- 支持多种瑕疵类型识别（划痕、污点、破损等）
- 可设置敏感度参数调整检测精度
- 支持瑕疵分类和统计

**参数配置**：
- **工具名称**：自定义工具标识
- **图像来源**：选择图像采集或上游工具输出
- **位置校准**：选择位置校准工具输出
- **ROI设置**：检测区域设置
- **瑕疵类型**：划痕/污点/破损/其他
- **敏感度**：高/中/低或数值调整
- **最小瑕疵面积**：过滤微小噪点
- **瑕疵分类规则**：基于形状、大小、灰度等
- **结果输出**：瑕疵数量、位置、大小、类型

**测试重点**：
- 不同类型瑕疵的检测准确性
- 敏感度参数对检测结果的影响
- 误报率和漏报率的平衡
- 不同材质表面的适应性
- 光照变化的鲁棒性

**常见问题**：
- 敏感度设置过高导致误报过多
- 敏感度设置过低导致漏检
- 相似颜色的瑕疵难以检测
- 复杂纹理背景干扰检测
- 动态范围不足导致暗区瑕疵漏检

### 2.3 位置校准工具
**功能描述**：
- 用于校准图像在采集过程中的位置偏移
- 支持旋转、缩放、平移等变换
- 基于基准图像进行位置对齐
- 提高后续检测的准确性

**参数配置**：
- **工具名称**：自定义工具标识
- **图像来源**：选择图像采集
- **基准图像**：选择已注册的基准图像
- **校准方法**：特征点匹配/模板匹配/边缘对齐
- **旋转校准**：启用/禁用，角度范围
- **缩放校准**：启用/禁用，缩放比例范围
- **平移校准**：启用/禁用，平移范围
- **匹配阈值**：特征匹配的置信度要求
- **结果输出**：校准后的图像、变换参数

**测试重点**：
- 不同角度旋转的校准准确性
- 不同缩放比例的校准准确性
- 复杂背景下的特征识别
- 部分遮挡情况下的校准能力
- 校准后的图像质量验证

**常见问题**：
- 特征点选择不当导致校准失败
- 环境变化导致特征点不可见
- 校准参数设置不当导致过度校正
- 计算资源消耗过大影响性能

### 2.4 2D/3D切换工具
**功能描述**：
- 用于在2D图像和3D图像之间进行切换和转换
- 支持3D点云数据的处理和分析
- 可提取高度、体积等3D特征
- 支持2D和3D数据的融合分析

**参数配置**：
- **工具名称**：自定义工具标识
- **数据来源**：2D图像/3D点云
- **转换模式**：2D→3D/3D→2D
- **高度映射方式**：灰度映射/颜色映射
- **分辨率设置**：点云密度调整
- **滤波参数**：噪声过滤
- **特征提取**：高度/体积/表面积/轮廓
- **结果输出**：转换后的图像或点云数据

**测试重点**：
- 2D和3D数据转换的准确性
- 高度信息提取的精度
- 不同分辨率下的性能和精度平衡
- 噪声过滤对数据质量的影响
- 复杂形状的处理能力

**常见问题**：
- 数据转换过程中的信息丢失
- 点云密度不足导致细节丢失
- 噪声过滤过度导致特征丢失
- 计算资源消耗过大
- 特殊材质表面的3D数据采集困难

- 结果的导出功能

**测试方法**：

- 结果导出格式验证

## 4. 测试数据管理

### 4.1 测试图像库
**推荐测试图像类型**：
- 标准参考图像（已知结果的图像）
- 边界条件图像（极小/极大特征）
- 复杂背景图像（干扰因素多）
- 低对比度图像（测试敏感度）
- 高噪声图像（测试抗干扰能力）
- 变形/缺陷样本图像（已知缺陷类型和位置）

**图像库组织**：
- 按测试目的分类
- 按难度级别分类
- 按工具类型分类
- 包含元数据（分辨率、光照条件、拍摄参数等）

### 4.2 测试用例管理
**测试用例分类**：
- 按功能模块分类
- 按测试类型分类
- 按优先级分类
- 按稳定性分类（回归测试用例）

**测试用例版本控制**：
- 与软件版本同步更新
- 记录测试用例的修改历史
- 追踪测试用例的执行情况
- 分析测试用例的有效性

## 5. 性能测试指标

### 5.1 响应时间
- 工具添加响应时间：<1秒
- 参数调整响应时间：<0.5秒
- 图像处理响应时间：<2秒（标准复杂度）
- 完整流程执行时间：<5秒
- 批量处理吞吐量：>10帧/秒

### 5.2 资源占用
- CPU使用率：峰值<80%
- 内存占用：<2GB
- 磁盘I/O：<10MB/s
- GPU使用率（如有）：峰值<90%

### 5.3 稳定性指标
- 连续运行时间：>24小时无异常
- 内存泄漏：无明显增长
- 崩溃率：<0.1%
- 异常处理成功率：>99%

## 6. 版本兼容性测试

### 6.1 向前兼容性
- 验证新版本软件能打开旧版本项目文件
- 验证旧版本的测试用例在新版本中有效
- 验证旧版本的配置参数在新版本中兼容

### 6.2 向后兼容性
- 验证新版本保存的项目文件能被旧版本打开（如需要）
- 验证新版本的测试结果格式与旧版本兼容
- 验证新版本的API接口与旧版本兼容

### 6.3 数据迁移
- 验证配置数据的迁移正确性
- 验证基准图像的迁移完整性
- 验证测试结果数据的迁移准确性
