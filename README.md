# 工业视觉软件测试用例生成Agent

## 项目概述

这是一个专为工业视觉软件测试设计的测试用例生成Agent，基于Cursor的Composer Agent模式实现。该Agent能够理解工业视觉软件的专业领域知识，自动生成结构化、专业、可执行的测试用例，帮助测试工程师提高测试效率和覆盖率。

## 功能特点

### 🔍 核心功能
- **智能测试用例生成**：基于软件功能描述自动生成测试用例
- **专业知识库支持**：内置工业视觉软件测试专业知识
- **多类型测试覆盖**：支持功能测试、参数测试、流程测试、兼容性测试、稳定性测试
- **模板化管理**：提供标准化的测试用例模板
- **测试覆盖度分析**：评估测试用例的覆盖情况

### 🎯 适用场景
- 工业视觉检测软件的功能测试
- 机器视觉算法的准确性验证
- 工业软件版本升级的回归测试
- 自动化测试脚本的基础用例生成
- 测试方案的快速规划和设计

## 快速开始

### 1. 环境准备

确保你已经安装了Cursor编辑器，并且了解Composer Agent的基本使用方法。

### 2. 配置Agent

1. 打开Cursor编辑器
2. 进入Composer Agent模式
3. 创建新的Agent配置
4. 复制以下文件内容到对应的配置区域：
   - `skill.md` → Skills配置
   - `rules.md` → Rules配置  
   - `prompt-templates.md` → Prompt Templates配置
   - `knowledge-base.md` → 导入到知识库

### 3. 使用方法

#### 基础使用

**生成特定工具的测试用例：**
```
我需要为面积检测工具生成标准级别的测试用例，包括功能测试和参数测试。

工具功能：用于检测图像中特定区域的面积大小，支持矩形、圆形、多边形ROI设置，可设置面积阈值进行合格/不合格判断。

请生成符合工业视觉软件测试标准的结构化测试用例。
```

**生成工作流程测试用例：**
```
我需要为工业视觉软件的完整工作流程生成测试用例。

工作流程：
1. 创建物料方案
2. 新建任务流程
3. 配置相机参数（CCD0）
4. 添加图像采集
5. 注册基准图像
6. 添加面积检测工具
7. 设置ROI和检测参数
8. 运行检测并验证结果

请生成验证整个流程正确性的测试用例。
```

**参数测试用例生成：**
```
我需要为瑕疵检测工具的参数配置生成测试用例。

工具参数列表：
- 敏感度：1-10级可调
- 最小瑕疵面积：1-1000像素
- 瑕疵类型：划痕/污点/破损/其他
- 阈值设置：0-255灰度值

请生成覆盖所有参数的边界值和典型值的测试用例。
```

## 知识库管理

### 知识库结构

知识库按以下结构组织：
1. **软件架构与功能模块**：整体架构和核心功能模块
2. **图像处理工具知识库**：各工具的详细信息
3. **通用测试方法**：工具操作、参数配置、ROI设置等测试方法
4. **测试数据管理**：测试图像库和测试用例管理
5. **性能测试指标**：响应时间、资源占用、稳定性指标
6. **版本兼容性测试**：向前/向后兼容性和数据迁移

### 知识库更新

建议定期更新知识库内容：
- 添加新工具的测试知识
- 更新测试方法和最佳实践
- 补充历史测试中发现的问题
- 完善性能测试基准数据

## 常见问题解答

### Q: 不同工具功能差异很大，需要为每个工具单独创建知识库吗？

**A:** 建议采用分层知识库结构：
- 通用层：包含所有工具共有的测试方法和原则
- 工具特定层：每个工具的专属知识
- 案例层：各工具的典型测试案例

这样既保证了知识的共享性，又能满足不同工具的特殊需求。

### Q: 如何保证生成的测试用例符合我们软件的实际情况？

**A:** 建议：
1. 提供详细的软件功能描述和操作流程
2. 分享现有的测试用例样本
3. 提供软件界面截图或操作视频
4. 定期反馈测试用例的使用情况，持续优化Agent

### Q: 如何处理版本升级时的测试用例更新？

**A:** 建议采用增量更新策略：
1. 识别版本间的功能差异
2. 为新增功能生成测试用例
3. 更新修改功能的测试用例
4. 保留未变更功能的测试用例

### Q: 生成的测试用例如何与自动化测试框架集成？

**A:** 可以：
1. 将测试用例导出为结构化格式（JSON/XML）
2. 基于测试用例生成自动化测试脚本模板
3. 建立测试用例ID与自动化脚本的映射关系
4. 使用测试用例管理工具进行集成

## 示例输出

### 面积检测工具测试用例示例

```markdown
## 测试用例：VIS-AREA-FUNC-001

### 基本信息
- **测试用例ID**：VIS-AREA-FUNC-001
- **测试用例名称**：矩形ROI面积检测功能验证
- **测试类型**：功能测试
- **优先级**：高
- **严重程度**：严重

### 测试环境
- **软件版本**：V2.0.0
- **硬件环境**：CCD0相机，Intel i7处理器，16GB内存
- **测试数据**：标准测试图像（包含已知面积的矩形区域）

### 前置条件
1. 软件已正常启动
2. 已创建测试项目和任务流程
3. 已配置CCD0相机并完成基准图像注册
4. 画布中显示清晰的测试图像

### 测试步骤
1. 在图像采集节点下添加"面积检测"工具
2. 在右侧参数面板中设置工具名称为"矩形面积检测"
3. 选择ROI类型为"矩形"
4. 在画布中绘制一个矩形ROI，覆盖测试图像中的已知面积区域
5. 设置面积计算方法为"像素计数"
6. 设置面积阈值：最小值100像素，最大值10000像素
7. 点击"应用"按钮保存配置
8. 点击"运行"按钮执行检测
9. 观察检测结果显示

### 预期结果
1. 工具添加成功，左侧工具树中显示"矩形面积检测"工具
2. ROI绘制成功，在画布中显示为绿色矩形框
3. 检测结果显示区域面积值，与已知标准值的误差在±2%以内
4. 结果状态显示为"合格"（在阈值范围内）
5. 工具运行过程无错误提示

### 实际结果
- [测试执行后的实际结果]

### 测试状态
- [通过/失败/阻塞]

### 备注
- 测试图像中的标准矩形区域面积为2500像素
- 可使用不同大小的矩形区域重复测试
```
